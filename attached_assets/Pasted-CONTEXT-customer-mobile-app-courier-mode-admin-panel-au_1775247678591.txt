PART 1 — STRIPE CONNECT EXECUTION LAYER
Реализуй production-ready integration layer для Stripe / Stripe Connect.

Что нужно:
1. Подготовить централизованный Stripe service layer.
2. Поддержать platform account logic для AIST.
3. Поддержать foundation для courier connected accounts:
   - Fleet courier
   - OSVČ courier
4. Подготовить account linking / onboarding foundation для курьеров.
5. Не ломать beta phase logic, где часть исполнения может идти как manual/controlled flow.
6. Если реальные keys/env ещё не даны:
   - сделай integration-ready code
   - чётко отдели что работает сразу, а что требует credentials

Нужно поддержать следующие операции:
1. PaymentIntent create
2. manual capture
3. cancel / release hold
4. partial refund
5. full refund
6. transfer / payout execution foundation
7. webhook handling

PART 2 — PAYMENT LIFECYCLE EXECUTION
Свяжи Stripe execution с order lifecycle.

Нужно:
1. При создании заказа поддержать правильный payment intent flow.
2. При cancel before dispatch:
   - release hold / cancel payment intent
3. При delivered:
   - capture funds
4. При SLA/refund decision:
   - создать реальный partial/full refund execution layer
5. При BFM cancel on video:
   - capture service fee only
   - release deposit hold foundation
6. Не дублировать финансовую логику по случайным местам.
7. Order должен хранить:
   - stripe ids
   - capture/refund state
   - payment state transitions
8. Все financial actions должны иметь audit trail.

PART 3 — INVOICE PDF GENERATION
Реализуй реальную генерацию faktura PDF на основе уже созданной invoice foundation.

Что нужно:
1. Перевести invoice generation из HTML foundation в реальный PDF workflow.
2. Использовать production-friendly renderer:
   - puppeteer
   - или другой надёжный HTML→PDF renderer
3. Генерировать PDF для faktura:
   - AIST OSVČ
   - neplátce DPH
   - invoice numbering без пропусков
4. Сохранять PDF в storage через подготовленный abstraction layer.
5. В invoice record хранить:
   - pdf_url
   - generated_at
   - delivery status
6. Не делать из этого идеальную бухгалтерскую типографию, но PDF должен быть usable и real.
7. Поддержать preview/download flow в admin и customer access layer.

PART 4 — EMAIL DELIVERY
Реализуй transactional email execution layer.

Нужно:
1. Подключить email provider abstraction:
   - SendGrid-ready preferred

2. Реализовать ключевые письма beta-фазы:
   - order accepted
   - courier assigned
   - delivered
   - invoice ready
   - refund/coupon issued
   - weekly courier statement
3. Не делать email marketing platform.
4. Только transactional email layer.
5. Если live credentials ещё нет:
   - сделать provider-ready integration
   - чётко отделить, что требует env keys
6. Логировать email attempts/status.
