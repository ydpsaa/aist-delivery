CONTEXT:
У проекта AIST уже есть рабочая структура:
- customer mobile app
- courier mode внутри мобильного приложения
- admin panel website
- auth
- realtime
- tracking
- admin dashboard

Сейчас не нужно строить новые большие функции. Нужно сделать полный визуальный ребрендинг и theme-system update под новый бренд AIST.

Главная цель:
1. Перевести интерфейс под новый логотип AIST.
2. Обновить цветовую палитру.
3. Сделать полноценную поддержку Light / Dark theme.
4. Обновить UI-стиль кнопок, карточек, акцентов и логотипов.
5. Сохранить текущую архитектуру и логику приложения.

BRAND DIRECTION:
Основой бренда является логотип AIST, который уже добавлен в проект.
Нужно использовать стиль:
- premium
- minimal
- modern logistics tech
- clean Apple-like UI
- soft glassmorphism / translucent elements
- rounded components
- дорогой и спокойный визуальный язык, без дешёвого неона

PRIMARY BRAND IDEA:
AIST = современный premium tech/logistics сервис.

COLOR DIRECTION:
Построй палитру на основе логотипа AIST:
- основной акцент: насыщенный синий / electric blue из логотипа
- deep blue / navy как вторичный акцент
- чёрный / почти чёрный для dark mode
- белый / светлый фон для light mode
- нейтральные серые для текстов, бордеров, secondary UI

Создай две полноценные темы:
1. Dark theme
2. Light theme

Плюс:
3. System theme support / theme switch support (архитектурно подготовить и внедрить)

DESIGN STYLE REQUIREMENTS:
1. Кнопки:
   - rounded
   - premium
   - semi-transparent / glass-like where appropriate
   - в духе последних Apple-style UI обновлений
   - primary CTA button, secondary outline/glass button, tertiary text button
2. Карточки:
   - мягкие скругления
   - светлая или тёмная полупрозрачная поверхность там, где это уместно
   - аккуратные тени / мягкие borders
   - premium dashboard / fintech style
3. Inputs:
   - clean
   - современный минимализм
   - правильно читаются в обеих темах
4. Navigation:
   - аккуратные active states
   - единый стиль табов, sidebar, headers
5. Status colors:
   - не ломать смысл статусов
   - success / warning / error должны остаться понятными
   - но привести их к новому брендовому окружению
6. Typography:
   - clean, premium, readable
   - визуально сильнее, чем сейчас
7. Логотип:
   - использовать icon-only и full logo version в правильных местах
   - не вставлять логотип хаотично
   - full logo использовать в splash, auth, admin login, key headers where needed
   - icon-only использовать в app icon areas / compact nav areas where appropriate
